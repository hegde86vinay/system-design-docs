RAG Bot — System Design

A fully local Retrieval-Augmented Generation pipeline for querying IPL 2025 player interviews, captain reactions, and post-match press conference content using natural language.

Platform macOS (M2 Air, 8 GB)
LLM Claude Haiku 4.5
Vector DB ChromaDB (local)
Embeddings all-MiniLM-L6-v2 (CPU)
Sources Indian Express · Times of India

📋Project Summary

This project is a hobby Gen-AI learning project that builds a Retrieval-Augmented Generation (RAG) pipeline on a local MacBook Air M2. It scrapes written IPL 2025 post-match content from public news sources, chunks and embeds those documents into a local ChromaDB vector store, and then uses the Claude Haiku API to answer natural-language questions grounded in that content.

The full pipeline runs entirely offline — no cloud vector DB, no external compute. The only network calls at query time are to the Claude API.

✂️② Chunker (chunker.py)

Loads every raw document, splits it into overlapping token windows, attaches the full metadata from the JSON sidecar, and writes all chunks to a single chunks.json file.

Chunking Strategy

ParameterValueWhy
Encodercl100k_base (tiktoken)Same tokeniser as Claude — accurate token counting
Chunk size500 tokensFits comfortably in Claude's context window while providing enough context per chunk
Overlap50 tokensPrevents important sentences being split across chunk boundaries
Minimum doc150 wordsAlready filtered at scrape time; chunker inherits this guard

Chunk Schema

{
  "id":           "a3f2c1b4e5d60789",   // SHA-256 of source_url::chunk_index
  "text":         "Virat Kohli said after the match...",
  "chunk_index":  0,
  "total_chunks": 4,
  "source_url":   "https://indianexpress.com/article/...",
  "title":        "Kohli on RCB's title win",
  "date":         "2025-06-03",
  "teams":        ["RCB", "PBKS"],
  "match_number": "",
  "author":       "",
  "source":       "indian_express"
}

🔢③ Embedder (embedder.py)

Loads chunks.json, converts each chunk's text to a 384-dimensional dense vector using all-MiniLM-L6-v2, and upserts into ChromaDB. Designed to be idempotent — safe to re-run without duplicating data.

Model Choice: all-MiniLM-L6-v2

PropertyValue
Dimensions384
Parameters~22M
Max sequence256 tokens
RAM footprint~90 MB
Speed (M2 CPU)~800 chunks/min
MTEB scoreStrong on semantic similarity tasks
Why not MPS (Apple GPU)? sentence-transformers has known compatibility issues with MPS on macOS. device="cpu" is explicit to prevent silent wrong results or crashes.

Idempotency

Each chunk's id is a 16-char SHA-256 of source_url::chunk_index. On re-run, ChromaDB's upsert is used — existing chunks are skipped, only new ones are added. This means you can run python embedder.py safely after scraping more articles without re-embedding everything.

🧠④ RAG Engine (rag.py)

The query engine: embeds the user's question, retrieves the top-3 most relevant chunks from ChromaDB, injects them as numbered context into a carefully crafted prompt, and calls the Claude API.

Prompt Design

SYSTEM:
"You are an expert IPL cricket analyst with access to IPL 2025 press
conference transcripts, match reports, and player interviews.
Answer questions using ONLY the provided context. If the context does
not contain enough information, say so clearly.
Always cite which match or interview your answer comes from."

USER:
Context [1]: <chunk text>  (source: Indian Express, 2025-05-14, teams: RCB)
Context [2]: <chunk text>  (source: Times of India, 2025-06-03, teams: RCB, PBKS)
Context [3]: <chunk text>  (source: Indian Express, 2025-06-03, teams: RCB)

Question: What did Virat Kohli say after RCB won the IPL 2025 final?

Public API

FunctionDescription
load_retriever()Initialises ChromaDB client + embedding model. Called once at startup.
retrieve(question, top_k=3)Embeds the question and returns top-K chunks with metadata.
build_prompt(question, chunks)Formats numbered context blocks + question into the final user message.
ask(question)Full pipeline: retrieve → build prompt → Claude API → return answer + sources.

Model: claude-haiku-4-5-20251001

Haiku was chosen over Sonnet for this project because query latency matters for a chat UI and the task (Q&A over retrieved context) doesn't need Sonnet-level reasoning. Cost is ~20× cheaper per token than Sonnet.

💬⑤ Streamlit UI (app.py)

A minimal but functional chat interface that wraps rag.py.

Features

  • st.chat_input — persistent text input, clears after each send
  • st.chat_message — alternating user/assistant bubbles with full history
  • Sources expander — after each answer: title, date, teams, URL per chunk
  • Sidebar stats — total docs, total chunks, collection name, model, last embed time
  • Session state — conversation history persists across reruns in the same session
  • Spinner — shows while Claude API call is in-flight

🤖GitHub Actions — CI Automation

Two chained workflows form an automated code review and implementation loop on every pull request to main.

flowchart TD PUSH["Developer pushes\nto feature branch"] --> PR["Pull Request\nopened / updated"] PR --> REV["claude-review.yml\n(pull_request trigger)"] REV --> BC{"Last commit\nauthor?"} BC -->|"human"| CREV["Call Claude API\nFresh-eyes review\n(diff only — no prior context)"] BC -->|"claude-implement-bot"| SKIP["Post: Human Review\nRequired — stop loop"] CREV --> RCOM["Post structured\nreview comment on PR"] RCOM --> IMP["claude-implement.yml\n(workflow_run trigger)"] IMP --> FETCH["Fetch review comment\nfrom PR via GitHub API"] FETCH --> IMPL["Call Claude API\nImplement Suggestions\n(code + review only)"] IMPL --> TEST["Run pytest tests/"] TEST -->|"pass"| COMMIT["Commit changes\nto feature branch"] TEST -->|"fail"| NOCOM["No commit\nlog failure"] COMMIT --> REPORT["Post implementation\nreport on PR"] NOCOM --> REPORT REPORT --> HUMAN["👤 Human reviews\nand approves"]

🔁Loop-Breaker Pattern

Without a guard, the implement bot's auto-commit would re-trigger the review workflow indefinitely. The loop-breaker prevents this:

How it works At the start of claude-review.yml, the last commit author is read via git log -1 --format='%an'. If it equals claude-implement-bot, all Claude API steps are skipped and a "Human Review Required" comment is posted instead. The chain stops there.
Commit Authorclaude-review behaviourclaude-implement behaviour
Human developerRuns full fresh-eyes reviewTriggers after review completes
claude-implement-botSkips review — posts human-review noticeDoes NOT re-trigger (workflow_run depends on review succeeding with a new review comment)

📖Run Book

First-time setup

# 1. Clone and install
git clone https://github.com/hegde86vinay/ipl-2025-pressconference-rag-bot
cd ipl-2025-pressconference-rag-bot
pip3 install -r requirements.txt

# 2. Add your Anthropic API key
cp .env.example .env
# edit .env → ANTHROPIC_API_KEY=sk-ant-...

Pipeline steps

  1. Scrape articles (~25–35 mins)

    python3 scraper.py — harvests 60–80 docs into data/raw/. Monitor data/scrape_log.txt for errors.

  2. Chunk documents (~10 secs)

    python3 chunker.py — splits docs into 500-token windows, writes data/chunks/chunks.json.

  3. Embed into ChromaDB (~5–8 mins)

    python3 embedder.py — encodes chunks with all-MiniLM-L6-v2 (CPU), upserts to vectorstore/chroma_db/.

  4. Launch chat UI

    streamlit run app.py — opens localhost:8501. Start asking questions.

⚠️Known Limitations

LimitationImpactMitigation
ESPNcricinfo & Cricbuzz blocked No match scorecards or official press conference transcripts Using Indian Express + TOI which carry quoted press conference excerpts
Content is 10+ months old (IPL 2025: Mar–Jun 2025) Some source URLs may have rotated or been archived Pagination + tag pages reach historical archive; re-scrape periodically
all-MiniLM-L6-v2 max sequence = 256 tokens Chunks longer than 256 tokens get silently truncated at embedding time 500-token chunks are text but ~200 unique tokens; acceptable for most press conf text
TOP_K = 3 chunks per query May miss relevant context for broad questions Configurable via config.TOP_K; increase to 5–8 for richer answers
No authentication on Streamlit app Anyone on the same network can access it Local development only; add Streamlit auth if exposed publicly
